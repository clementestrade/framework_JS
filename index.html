Rapport projet Framework JS

Introduction :

Dans le cadre de la licence professionnel LPDW, nous avons dû réaliser un site pour des apiculteurs en utilisant l’un des framework JS vu en cours. Nous avons choisi d’utiliser le framework View JS car ils nous paraissaient plus facile d’utilisation.

1. L’organisation du projet

Après réception du projet nous nous sommes d’abord concertés afin de se mettre d’accord sur la vision du projet pour cela des maquettes ont été réaliser individuellement puis mise en commun afin de s’assurer d’une vision commune du projet. Nous nous sommes ensuite réparti les tâches de manière équitable en mis en place un GIT afin de simplifier la mise en commun.

2. Charte Graphique

Pour l’aspect graphique nous avons choisi d’utiliser la couleur jaune afin de rappeler les abeilles ainsi qu’un gris pour entrer en contraste avec le jaune et simplifier la lecture, le bouton et lui en vert afin de rappeler la nature proche aux métiers d’apiculteurs.

3. Organisation des pages

Notre site contient trois pages plus la page d’accueil :

· Une liste des rucher avec la possibilité de lister les ruchers, en ajouter un et effectuer une visite.

· Une carte répertoriant les différents rucher en fonction de leurs coordonnées.

· Une page paramètre pour modifier les paramètres par défaut.

4. Indexed DB

Pour faire persister les données dans le site on ne pouvait pas utiliser de réelle base de données puisque toute l’application devait être réaliser en JS sans autre langage, nous avons donc fait le choix d’utiliser l’indexedDB contenue dans chaque navigateur ainsi les données sont persistante tant qu’on utilise le même navigateur.

Notre base de données comporte ainsi deux tables :

· Une table rucher avec un identifiant, son nom le nombres de ruche, un descriptif, les coordonnées géographique, une date de création ainsi qu’une fréquence de visite.

· Une table visite relier à la table rucher grâce à l’identifiant et possédant en plus les champs que l’utilisateur peut saisir telle qu’une description d’une visite de la nourriture ou du nombre de récolte.

Conclusion

Ce projet n’est pas entièrement terminé en laisse voir de nombreuse possibilité d’amélioration par rapport au sujet donné initiale, cependant aux vues de la charge de travail en entreprise et du temps impartis relativement cours nous n’avons pas réussi à lui consacrer le temps nécessaire pour mener ce projet à bien. Il nous à néanmoins permis de comprendre l’intérêt de l’utilisation d’un framework JS par rapport au JS natif en prenant conscience de la simplification.
